1Programlama Dillerine Giriş

🎯 Neden Programlama Dillerini Öğreniriz?

  • Farklı dillerin güçlü/zayıf yönlerini anlamak
  • Yeni dilleri daha kolay öğrenmek
  • Algoritma ifade kabiliyeti artar
  • Dil tasarımı ve uygulaması için temel oluşturur

📊 Dil Tasarımı Kriterleri

✅ Okunabilirlik (Readability)

  • Açık, anlaşılır sözdizimi
  • Minimal anahtar kelimeler
  • Yapısal programlama desteği

✍️ Yazılabilirlik (Writability)

  • Soyutlama desteği
  • İfade gücü
  • Tip kontrolü esnekliği

🛡️ Güvenilirlik (Reliability)

  • Tip denetimi
  • İstisna yönetimi
  • Aliasing kontrolü

⚙️ Maliyet (Cost)

  • Eğitim maliyeti
  • Yazım/çalıştırma maliyeti
  • Bakım maliyeti

⚙️ Derleme Süreci

Kaynak Kod → Lexer (Sözcüksel Analiz) → Parser (Sözdizimsel Analiz) → Semantik Analiz → Kod Üretimi → Çalıştırılabilir
  • Lexer/Scanner: Kaynak kodu token'lara böler (anahtar kelimeler, tanımlayıcılar, sabitler)
  • Parser: Token dizisini gramer kurallarına göre ağaca dönüştürür (Parse Tree / AST)
  • Semantik Analiz: Tip kontrolü, kapsam kontrolü
  • Kod Üretimi: Hedef dil/makine kodu üretilir
⚠️ Sınav Sorusu: Compiler vs Interpreter farkı? Compiler tüm kodu önceden çevirir. Interpreter satır satır çalışır. JVM ikisini birleştirir (bytecode + JIT).

2Sözdizimi – BNF, EBNF ve Parsing

📐 BNF (Backus-Naur Form)

  • Programlama dillerinin sözdizimini tanımlamak için kullanılan biçimsel notasyon
  • Terminal: Değiştirilemez semboller (gerçek kelimeler/karakterler)
  • Non-terminal: Başka kurallara genişletilebilir semboller (<expr>, <stmt>)
  • Üretim Kuralı: <A> ::= <B> | <C> (A ya B ya da C olabilir)
BNF Örnek:
<expr> ::= <expr> + <term> | <term>
<term> ::= <term> * <factor> | <factor>
<factor> ::= ( <expr> ) | id | number

📐 EBNF (Extended BNF)

  • [ ] → İsteğe bağlı (0 veya 1 kez)
  • { } → 0 veya daha fazla kez tekrar
  • ( ) → Gruplama
  • BNF'e göre daha okunabilir ama eşdeğer güce sahip

🌲 Parse Tree vs AST

Parse Tree (Ayrıştırma Ağacı)

  • Tüm gramer sembolleri dahil
  • Her terminal ve non-terminal var
  • Çok detaylı

AST (Abstract Syntax Tree)

  • Gereksiz detaylar ayıklanmış
  • Sadece anlamsal yapı
  • Derleyicilerde kullanılır

🔍 Parsing Teknikleri

TeknikYönAçıklama
Top-Down (LL)↓ Kök→YaprakBaşlangıç sembolünden türetme, recursive descent
Bottom-Up (LR)↑ Yaprak→KökShift-reduce, daha güçlü

⚠️ Ambiguity (Belirsizlik)

Bir gramer, aynı string için birden fazla parse tree üretiyorsa belirsiz (ambiguous)tir. Belirsizlik önemli çünkü hangi anlama geleceği belli değil.

Çözüm: Gramer yeniden yazılır VEYA öncelik/ilişkisellik kuralları eklenir.

🔢 Operatör Önceliği ve İlişkisellik

  • Precedence (Öncelik): Hangi operatör önce değerlendirilir? (* önce +)
  • Associativity (İlişkisellik): Aynı öncelikteki operatörler hangi yönden? Sol ilişkili: a-b+c → (a-b)+c
  • Sağ ilişkili örnek: 2**3**2 → 2**(3**2) = 512
✅ Hatırlatma: ALGOL 60 dili, BNF notasyonunu ilk kullanan dildir. John Backus ve Peter Naur tarafından geliştirilmiştir.

3Bağlama (Binding) ve Yaşam Süresi

🔗 Binding Nedir?

Bir ismin (değişken, fonksiyon) bir özelliğe (tip, değer, adres) bağlanmasıdır. Ne zaman bağlandığı çok önemlidir.

⏱️ Binding Zamanları

ZamanNe Zaman?Örnek
Dil tasarım zamanıDil tasarlanırken+, * operatörlerinin anlamı
Derleme zamanıDerleme sırasındaC'deki int tipi → 4 byte
Bağlama zamanıLink aşamasındaKütüphane fonksiyonları
Yükleme zamanıProgram belleğe yüklenirkenGlobal değişken adresi
Çalışma zamanıProgram çalışırkenDinamik değişkenler

📌 Statik Bağlama

  • Çalışma zamanından önce bağlanır
  • Değişmez
  • Daha hızlı, güvenli
  • Örnek: C'de int x = 5;

🔄 Dinamik Bağlama

  • Çalışma zamanında bağlanır
  • Değişebilir
  • Daha esnek ama yavaş
  • Örnek: Python'da x = 5

📦 Değişken Özellikleri (5 Temel)

  • 1. İsim (Name): Değişkene verilen etiket
  • 2. Adres (Address/l-value): Bellekteki konumu
  • 3. Değer (Value/r-value): İçerdiği veri
  • 4. Tip (Type): Değerin türü ve işlem kuralları
  • 5. Yaşam Süresi (Lifetime): Bellekte var olduğu süre

🕐 Değişken Yaşam Süreleri (Storage Classes)

TürNe Zaman Oluşur?Ne Zaman Yok Olur?Örnek
StatikProgram başındaProgram biterC static değişken
Stack-DynamicFonksiyon çağrıldığındaFonksiyon biterLocal değişkenler
Heap-Dynamic (Explicit)malloc/new ilefree/delete ileC: malloc, C++: new
Heap-Dynamic (Implicit)Atama ileGarbage collectorJava, Python nesneleri
⚠️ Sınav İpucu: Aliasing = Aynı bellek adresine birden fazla ismin bağlı olması. Güvenilirliği düşürür çünkü bir isimden yapılan değişiklik diğerini de etkiler.

4Alt Programlar (Subprograms)

Temel Kavramlar

  • Fonksiyon: Değer döndürür
  • Prosedür/Void: Değer döndürmez, yan etki üretir
  • Formal Parameter: Tanımda kullanılan parametre ismi
  • Actual Parameter (Argument): Çağrıda gönderilen gerçek değer

Parametre Geçiş Yöntemleri

YöntemNasıl ÇalışırDiller
Pass by ValueDeğerin kopyası geçilir. Orijinal değişmez.C (default), Java (primitives)
Pass by ReferenceDeğişkenin adresi geçilir. Orijinal değişebilir.C++ (&), Fortran
Pass by Value-ResultGiriş kopyalanır, çıkışta geri yazılır.Ada (in out)
Pass by Nameİfade çağrı anında değerlendirilir (lazy).Algol 60
Sınav İpucu: Pass by Value vs Reference farkı!
void f(int x){x=10;} → By Value: orijinal DEGISMEZ
void f(int &x){x=10;} → By Reference: orijinal DEGISIR

Closure (Kapatma)

Bir fonksiyonun, tanımlandığı ortamdaki (lexical scope) değişkenlere erişimi korunur. Fonksiyon başka bir yerde çağrılsa bile o değişkenlere ulaşabilir. Python, JavaScript, F# closure destekler. C desteklemez.

Coroutine

  • Normal fonksiyonlar: Çağrılır, biter, geri döner
  • Coroutine: Birden fazla giriş/çıkış noktası var, state korunur
  • yield ile askıya alınır, resume ile devam eder
  • Python generator fonksiyonları coroutine örneğidir

Activation Record (Çağrı Kaydı)

  • Her fonksiyon çağrısında bir Activation Record (AR) stack üzerinde oluşturulur
  • AR içeriği: yerel değişkenler, parametreler, dönüş adresi, dynamic link, static link
  • Dynamic Link: Çağıran fonksiyonun AR'ına işaret eder
  • Static Link: Statik ebeveyn fonksiyonun AR'ına işaret eder (lexical scope)

5Kapsam (Scope) ve Görünürlük

Kapsam Nedir?

Bir değişkenin programın hangi bölümlerinden görülebileceğini (erişilebileceğini) belirler.

Statik Kapsam (Lexical Scope)

  • Kapsam, kaynak kodun yapısına göre derleme zamanında belirlenir
  • C, Java, Python, C++ bu yöntemi kullanır
  • İç blok, dış bloktaki değişkeni görebilir

Dinamik Kapsam (Dynamic Scope)

  • Kapsam, çalışma zamanındaki çağrı zincirine göre belirlenir
  • Hangi fonksiyon çağırdıysa, onun değişkenleri erişilebilir olur
  • Lisp (eski), Perl dinamik scope kullanır
  • Hata bulmayı zorlaştırır

Statik Scope Örneği

x = 10 (global)
def f(): x görülür → 10
def g(): x=20; f() → f hala 10 görür

Dinamik Scope Örneği

x = 10 (global)
def f(): x görülür
def g(): x=20; f() → f 20 görür!

6Veri Tipleri ve Tip Denetimi

Temel Veri Tipleri

  • Integer: Tam sayı. C: int(4B), short(2B), long(8B)
  • Float: IEEE 754. float(4B), double(8B)
  • Boolean: true/false. C'de yoktu: 0=false, sıfır-dışı=true
  • Char: ASCII veya Unicode
  • String: Java: immutable (değiştirilemez)

Yapısal Tipler

  • Array: Aynı tipte elemanlar. Statik veya dinamik boyut
  • Record/Struct: Farklı tipte alanlar. Ada: record, C: struct
  • Tuple: Sıralı, sabit sayıda eleman. Python: (1, "a", 3.14)
  • Union: Aynı bellekte farklı tipler. C: union
  • Discriminated Union: Hangi tip aktif bilgisi de tutulur (güvenli)

Pointer Problemleri

  • Dangling Pointer: Serbest bırakılmış belleği gösteren pointer → HATA KAYNAĞI
  • Memory Leak: Ayrılan bellek hiç serbest bırakılmamış → KAYNAK İSRAFI

Tip Denetimi

Statik Tip Denetimi

Derleme zamanında kontrol.
Hızlı çalışma. C, Java, C++

Dinamik Tip Denetimi

Çalışma zamanında kontrol.
Esneklik. Python, JavaScript, Ruby
Güçlü Tip Denetimi (Strong Typing): Tüm tip hatalarının tespit garantisi.
Java ve Ada: Güçlü tipli. C/C++: Tam değil (union, pointer casting sorun çıkarır).

Tip Uyumluluğu ve Eşdeğerliği

KavramAçıklama
CompatibilityCoercion ile kullanılabilir (esnek)
EquivalenceCoercion olmadan değiştirilebilir (katı)
Name EquivalenceAynı isimle tanımlı ise eşdeğer (çok katı)
Structure EquivalenceYapıları aynıysa eşdeğer (esnek ama riskli)

Tür Dönüşümleri

  • Widening: int → float. Güvenli, veri kaybı yok
  • Narrowing: double → int. VERİ KAYBI OLABİLİR!
  • Coercion: Derleyicinin otomatik yaptığı örtük dönüşüm
  • Explicit Cast: (int)x — programcı tarafından açıkça yapılır

7İfadeler, Operatörler ve Atama

Aritmetik İfadeler

  • Bileşenler: Operatörler, operandlar, parantezler, fonksiyon çağrıları
  • Unary: -a, Binary: a+b, Ternary: a?b:c

Operatör Önceliği (Genel)

1. Üs alma (**)   2. Çarpma/Bölme   3. Toplama/Çıkarma

İlişkisellik (Associativity)

  • Sol ilişkili: a - b + c → (a-b)+c (çoğu operatör)
  • Sağ ilişkili: 2**3**2 → 2**(3**2) = 512 (üs alma)
  • Ada: A**B**C → HATA verir (belirsizlik)

Operatör Aşırı Yükleme (Overloading)

  • + operatörü: int toplama, float toplama, string birleştirme (Java)
  • C++, C#, F# kullanıcı tanımlı overloading destekler
  • Java desteklemez
  • Avantaj: Okunabilirlik. Dezavantaj: Aşırı kullanımda karmaşa

Referential Transparency

Aynı değere sahip ifadeler birbirinin yerine geçebiliyorsa referans şeffaflığı var. Fonksiyonel dillerde yan etki olmadığından bu sağlanır. Kolayca analiz ve test edilir.

Short-Circuit Değerlendirme

  • Tüm operandları değerlendirmeden sonuç belirlenir
  • (a >= 0) && (b < 10) → a < 0 ise ikinci ifade DEĞERLENDİRİLMEZ
  • C: && ve || short-circuit; & ve | değil
  • Ada: and then / or else (short-circuit), and / or (değil)

8Kontrol Akisi ve Secim Yapilari

Kontrol Akisi Seviyeleri

  • Ifade duzeyi: operator onceligi ve iliskisellik
  • Deyimler arasi: if, loop gibi yapilar
  • Program birimleri arasi: fonksiyon/prosed ur cagrilari

Secim Yapilari (if-else)

  • Ikili secim: if...else
  • Coklu secim: switch-case
  • Dangling Else Problemi: else en yakin if'e baglanir (Java/C)
  • Python'da girintiye gore belirlenir
  • Ruby'de end kullanildigi icin belirsizlik olmaz

Switch-Case Ozellikleri

  • C: integer, char gibi discrete type kullanilir
  • case degerleri sabit olmali
  • Fall-through: break konulmazsa bir sonraki case'e duser!
  • break: switch'ten hemen cikmak icin kullanilir
Sinav Ipucu — Fall-through: C'de break olmadan case bloklari birbirine akar. switch(x){case 1: odd++; case 2: even++;} → x=1 ise HER IKISI calisir!

Fonksiyonel Dillerde if

Fonksiyonel dillerde if bir deyim (statement) degil, bir ifade (expression)'dir. Yani bir deger dondurur. else zorunludur! y = if x > 0 then 1 else -1

9Dongular ve Iterasyon

Dongu Cesitleri

TurKontrolOrnek
Counter-controlledSayac ilefor (i=0; i<10; i++)
Logical-controlledMantiksal kosul ilewhile (x > 0)
Data-basedKoleksiyon uzerindefor x in liste

Pretest vs Posttest

Pretest (while)

Dongu BASLAMADAN kontrol yapilir.
Kosul false ise HIC CALISMAZ.
while(x > 0) {...}

Posttest (do-while)

Dongu EN AZ 1 KEZ calisir.
Sonra kontrol edilir.
do {...} while(x > 0);

Sayacli Dongu Tasarimi

  • 3 temel parametre: baslangic degeri, bitis degeri, artis miktari (step)
  • Dongu degiskeni tip secimi: int, float, char?
  • Dongu icinde degistirilebilir mi? (Cogu dil izin vermez)

Fonksiyonel Dillerde Dongu

Fonksiyonel dillerde dongu YOKTUR. Bunun yerine recursion (ozyineleme) kullanilir. Tail-recursion: Son cagri ozyinelemeyse, optimize edilebilir (stack overflow olmaz).

10Hesap Kurami ve Halting Problemi

Hesap Edilebilirlik (Computability)

  • Bir problemin bilgisayar tarafindan cozulebilip cozulemedigi
  • Alan Turing (1936): Evrensel Turing Makinesi modeli
  • Bazi problemler hesaplanamazdir (undecidable)

Halting Problemi (Durma Problemi)

Tanim: Bir program ve giris verildiginde, programin sonunda durup durmayacagini onceden soylemek mumkun mudur?

Cevap: HAYIR — Bu problem cozulemezdir (undecidable).
  • Turing 1936'da kantlamistir
  • Kanit yontemi: Celisme (contradiction) ile
  • Bir duran-mi? fonksiyonu yazarsak, onu kendine uygularsak celiski cikar

Rice Teoremi

  • Bir programin davranisi hakkindaki her non-trivial ozellik hesaplanamazdir
  • Ornegin: "Bu program bir string mi uretiyor?" sorusu bile hesaplanamazdir

Dilerin Gucuyle Ilgili

  • Turing-Complete dil: Herhangi bir hesaplamali problemi cozebilir
  • Cogu modern programlama dili Turing-Complete'tir
  • HTML/CSS Turing-Complete degil (hesaplama yapamaz)

11Sik Sorulan Sorular ve Ozet

Sik Sinav Sorulari

  • Coercion nedir? Derleyicinin otomatik yaptigi ortuk tur donusumu (int → float)
  • Strong typing nedir? Tum tip hatalarinin tespit edilme garantisi
  • Dangling pointer nedir? Serbest birakilmis bellegi gosteren pointer
  • Closure nedir? Fonksiyonun tanimlandigi ortamin degiskenlerine erisimi
  • Aliasing nedir? Ayni bellek adresine birden fazla ismin baglanmasi
  • Referential transparency? Ayni deger = birbirinin yerini alabilir (yan etki yok)
  • Short-circuit? Gerek yoksa ikinci operand degerlendirilmez
  • Halting problemi? Bir programin durup durmayacagi onceden soylenemez
  • Static vs Dynamic scope? Statik: kod yapisina gore. Dinamik: cagri zincirine gore
  • Deep vs Shallow binding? Deep: tanimlama aninda. Shallow: cagri aninda
